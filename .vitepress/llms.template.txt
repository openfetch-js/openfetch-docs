## Install & import

```bash
npm install @hamdymohamedak/openfetch
```

**Core client**

```ts
import openFetch, {
  create,
  createClient,
  createRetryMiddleware,
  createCacheMiddleware,
  MemoryCacheStore,
  OpenFetchError,
  isOpenFetchError,
  assertSafeHttpUrl,
} from "@hamdymohamedak/openfetch";
```

**Tree-shakable subpaths (v0.2.8+)**

```ts
import { retry, timeout, hooks, debug, strictFetch } from "@hamdymohamedak/openfetch/plugins";
import { createFluentClient } from "@hamdymohamedak/openfetch/sugar";
```

- Default export `openFetch` = `createClient()` with no initial defaults.
- `create` is an alias of `createClient`.

## Client API (concise)

- `createClient(initialDefaults?)` — returns `OpenFetchClient`.
- Methods (merge `defaults` + per-call config):
  - `get(url, config?)`, `head`, `options` — optional `params` → query string.
  - `post(url, data?, config?)`, `put`, `patch` — optional `data`; plain objects → JSON + `content-type: application/json` unless already set; `FormData` / `URLSearchParams` / `Blob` / binary views are not forced to JSON.
  - `delete(url, config?)` — no `data` argument; use `config.data` / `config.body` if the API needs a body.
  - `request(urlOrConfig, config?)` — full `method`, `responseType`, `rawResponse`, etc.
- `client.use(middleware)` — appends to `defaults.middlewares`.
- `client.interceptors.request.use` / `response.use` — request stack **LIFO**, response **FIFO**.
- Throws if merged config has no resolvable `url` (message includes ``openfetch: `url` is required``).

## Response shape

- Default (`unwrapResponse` falsy): `Promise<OpenFetchResponse<T>>` → `{ data, status, statusText, headers, config }`.
- `unwrapResponse: true`: `Promise<T>` (body only).
- **`rawResponse` / fluent `.raw()`** (v0.2.8+): native `Response` as `data` without adapter body parsing on that path; client response interceptors still run.

## Config highlights

- `baseURL`, `url`, `params`, `headers`, `timeout`, `signal`, `data` / `body`, `auth` (Basic), `withCredentials`, `responseType` (`json`|`text`|`blob`|`arraybuffer`|`stream`), `rawResponse`, `validateStatus` (default 2xx), `transformRequest` / `transformResponse` (arrays concatenated), `middlewares`, `retry`, `memoryCache`, plus `RequestInit` passthrough fields.

## Errors

- `OpenFetchError` with `code`: `ERR_BAD_RESPONSE`, `ERR_NETWORK`, `ERR_PARSE`, `ERR_CANCELED`.
- Use `error.toShape({ includeResponseData: false, includeResponseHeaders: false })` before client-facing or shared logs.

## Middleware & plugins

- **`createRetryMiddleware()`** — exponential backoff; by default **does not** retry non-idempotent methods for network/status retries unless `retryNonIdempotentMethods: true`. Optional total / per-attempt timeouts, idempotency key helpers for retried POSTs.
- **`createCacheMiddleware(store, options)`** — GET/HEAD in-memory; use `varyHeaderNames` or custom `key` when responses vary by auth.
- **`@hamdymohamedak/openfetch/plugins`** — `retry()`, `timeout()`, `hooks()`, `debug()` (e.g. masked headers), `strictFetch()`.
- **`@hamdymohamedak/openfetch/sugar`** — `createFluentClient()`: URL + method chaining, terminals `.json()` / `.text()` / `.send()` / `.raw()`, optional `.memo()` (one round-trip, buffered body — not HTTP cache).

## Security

- `assertSafeHttpUrl(url)` — optional guard for untrusted URLs (http/https; blocks some literal private/loopback IPs). **Not** a full SSRF defense when hostnames are attacker-controlled (DNS rebinding); pair with allowlists / network policy when relevant.

## Framework usage (short)

- React: `useEffect` + `openFetch.get(fullUrl)`; JSON → `res.data`; cancel on unmount with `AbortController` / flag when needed.
- Vue 3: `onMounted` + `openFetch.get(fullUrl)` → `ref.value = res.data`.
- RSC: `await openFetch.get(fullUrl)` in an async server component when supported.
- Examples: `/framework-guides` (English), `/ar/framework-guides` (Arabic).
